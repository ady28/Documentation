Target: windows server 2019

works also on refs

type:
 general purpose fs
 hyper-v workload
 backup target 

steps:
 scans the fs for files meeting the optimization  policy
 breaks files into variable sized chinks
 identify unique chunks
 place chunks in the chunk store (can also compress them)
 

(dedups the first 1 TB of a file for files larger than 1 TB)

tasks:
 optimization (check files and create chunks)
 garbage collection (reclaim space)
 integrity scrubbing (important for storage spaces and sp direct on ReFS) - found in the Data Integrity scheduled tasks folder

DDPEval.exe to evaluate the savings
